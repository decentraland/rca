# March 14th, 2026 - Users unable to login with Crypto wallets / Magic options

|                        |               |
| ---------------------: | :------------ |
|           **Ticket ID** | DCL-2245      |
|  **Reported on Slack** | March 14, 2026 |
| **Solution Completed** | March 14, 2026 |

## What happened?

Users were unable to log in to the Explorer or other Decentraland sites using Crypto wallets or Magic link options. Only Email+OTP login remained functional. The Explorer screen loaded but displayed a connection error when attempting login. The root cause was a stale container image digest: the auth-server Fargate service on the prd-main cluster was pinned to a "latest" tag digest (`sha256:fdd5bb70a1e7…`) that had been garbage-collected from the quay.io container registry after the tag was moved to a newer image on March 9th.

## Why did it happen?

The auth-server ECS task definition referenced the image `quay.io/decentraland/auth-server:latest` with a pinned digest (`sha256:fdd5bb70a1e7…`). On March 9th, the "latest" tag was re-tagged multiple times to newer image digests (`36b75f15a549`, `a05c91997465`, `3f3efbd039c1`) as part of routine releases. Once no tag pointed to the old digest, quay.io's garbage collection removed the manifest. When Fargate attempted to pull the image using the cached digest on March 14th during a AWS maintenance, the registry returned "not found," causing the `CannotPullContainerError` and preventing the auth-server from starting.

The underlying issue is that the existing GitHub CI/CD workflow pushes mutable "latest" tags, and ECS caches the digest at deploy time. When the tag moves and the old manifest is garbage-collected, any service that needs to re-pull the image (e.g., during scaling events or task restarts) will fail.

## Impact

- **Severity**: SEV-1 – Critical user-facing outage.
- Users were unable to log in using Crypto wallets or Magic link options. Only Email+OTP remained functional.
- The Explorer displayed a connection error upon login attempt.

## Solution

All Fargate services running on the prd-main cluster with mutable "latest" image tags were identified and re-deployed using the equivalent pinned release version tag or image digest hash, restoring normal functionality.

## Preventive Actions

- **Alert** *(pending)* – Deploy a new Lambda to monitor all ECS task definitions on prd-main and ensure no services reference "latest" tags with deleted digests. Report any findings to Slack.
