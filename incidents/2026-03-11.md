# March 11th, 2026 - Profile dapp wallet connection broken due to thirdweb dependency mismatch

|                        |               |
| ---------------------: | :------------ |
|  **Reported on Slack** | March-11-2026 |
| **Solution Completed** | March-11-2026 |

## What happened?

The Profile dapp broke at runtime when trying to connect a wallet. Users encountered the following error:

```
TypeError: J.createThirdwebClient is not a function
    at ThirdwebConnector.getClient (ThirdwebConnector.js:65:32)
    at ThirdwebConnector.activate (ThirdwebConnector.js:106:35)
    at ConnectionManager.connect (ConnectionManager.js:30:48)
    at ConnectionManager.tryPreviousConnection (ConnectionManager.js:83:54)
```

This prevented users from logging in or reconnecting their wallets to the Profile dapp.

## Why did it happen?

A transitive dependency version mismatch between `decentraland-connect` v11.x and the thirdweb SDK. The `ThirdwebConnector` in `decentraland-connect` called `createThirdwebClient()`, a function from the thirdweb SDK, but the version of the thirdweb SDK resolved in the Profile dapp's dependency tree did not export that function. This incompatibility caused the runtime `TypeError` when the connector tried to initialize the thirdweb client during wallet activation.

## Issue Timeline - UTC

1. 2026-03-11 - Runtime error reported: wallet connection failing on Profile dapp with `createThirdwebClient is not a function`
2. 2026-03-11 - Root cause identified as a dependency version mismatch in `decentraland-connect` and thirdweb SDK
3. 2026-03-11 - Fix deployed via [PR #434](https://github.com/decentraland/profile/pull/434) bumping `decentraland-connect` from v11.2.0 to v12.0.1 and `decentraland-dapps` from v26.10.0 to v28.2.1

## Impact

- **Severity**: SEV-2
- Users were unable to connect their wallets or log in to the Profile dapp.

## Solution

Bumped `decentraland-connect` from `^11.2.0` to `^12.0.1` and `decentraland-dapps` from `^26.10.0` to `^28.2.1` in [PR #434](https://github.com/decentraland/profile/pull/434). The major version bumps resolved the transitive dependency tree, pulling in a compatible version of the thirdweb SDK that correctly exports `createThirdwebClient`.
