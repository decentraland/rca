# Mar-04-2026 - Unauthenticated newsletter unsubscription of other users

|                          |              |
| -----------------------: | :----------- |
| **Reported on Immunefi** | Mar-04-2026  |
|           **Mitigation** | Mar-04-2026  |
|   **Solution Completed** | Mar-04-2026  |

## Description

Builder API exposed newsletter management endpoints without authentication or
ownership checks. `POST /v1/newsletter` returned a stable `subscriptionId` for
any supplied email. Because the same email always resolved to the same id, an
attacker could collect that identifier and immediately call
`DELETE /v1/newsletter/:subscriptionId` to remove another user's subscription —
no login, wallet signature, or victim interaction required.

## Severity

Websites & Applications - Low

## Solution

The `subscriptionId` is no longer included in the response body of
`POST /v1/newsletter`; only a success or error status is returned. The
`GET /v1/newsletter/:subscriptionId` and `DELETE /v1/newsletter/:subscriptionId`
endpoints were removed entirely, as they were no longer needed.
