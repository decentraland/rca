# April 7, 2026 - Any Authenticated User Can Modify or Delete Any Event

|                          |               |
| -----------------------: | :------------ |
| **Reported on Immunefi** | April-04-2026 |
|           **Mitigation** | April-07-2026 |
|   **Solution Completed** | April-07-2026 |

## Description

- The `PATCH /api/events/:event_id` endpoint in the Events API (`events.decentraland.org`) applied editable attributes — including `name`, `description`, `image`, `start_at`, `duration`, `x`, `y`, and `rejected` — from the caller's request body before performing any ownership check, allowing any authenticated wallet (including zero-balance throwaway wallets) to overwrite fields on events they do not own.
- Setting `rejected: true` permanently deletes any event, as rejected events are filtered from all API responses and cannot be recovered via the API.

## Severity

Websites & Applications - Critical.

## Solution

[Move the ownership check before the attribute assignment](https://github.com/decentraland/events/commit/1546a9fa5ebdea07a292e528413c5192830e35cc) so that editable fields from the request body are only applied after verifying the caller owns the event.
